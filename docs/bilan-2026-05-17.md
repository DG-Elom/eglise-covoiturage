## 11. Règles métier déjà en place

- **Auto-annulation cross-instance** : quand un conducteur accepte une demande, toutes les autres demandes pending du même passager pour le même culte+date+sens (chez d'autres conducteurs) sont automatiquement annulées avec motif `accepte_ailleurs`. Implémenté dans [`/api/reservations/[id]/accept/route.ts`](../src/app/api/reservations/[id]/accept/route.ts).
- **Garde-fou capacité** : trigger DB `check_instance_capacity` (BEFORE INSERT v29) + `check_instance_capacity_on_update` (BEFORE UPDATE v37) — empêche tout surnominage.
- **Auto-refus de la file d'attente** : quand un conducteur accepte et que l'instance devient pleine, les autres pending de cette instance sont auto-refusées avec motif `trajet_complet_auto` (trigger v29).
- **Suggestion d'alternatives** : quand une demande échoue pour cause de trajet plein, le serveur renvoie les 3 trajets compatibles les plus proches.
